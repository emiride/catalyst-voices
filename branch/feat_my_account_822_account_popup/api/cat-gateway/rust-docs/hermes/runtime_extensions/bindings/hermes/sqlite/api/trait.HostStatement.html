fn finalize(&mut self, self_: Resource<Statement>) -> Result<Result<(), Errno>>

Destroys a prepared statement object. If the most recent evaluation of the statement encountered no errors or if the statement is never been evaluated, then the function results without errors. If the most recent evaluation of statement failed, then the function results the appropriate error code.

The application must finalize every prepared statement in order to avoid resource leaks. It is a grievous error for the application to try to use a prepared statement after it has been finalized. Any use of a prepared statement after it has been finalized can result in undefined and undesirable behavior such as segfaults and heap corruption.
